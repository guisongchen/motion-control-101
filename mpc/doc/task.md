Week 9 实验方案：OSQP + MuJoCo MPC

实验模型：MuJoCo cart-pole（或自建平面倒立摆）。选择 cart-pole 是因为它是经典的欠驱动系统，与 LIPM 有相似的结构（基座驱动 + 摆杆自由运动），且 MuJoCo 原生支持。

---
实验环境与依赖

| 依赖 | 版本/说明 |
|------|-----------|
| Python | 3.10+ |
| MuJoCo | 3.x（mujoco pip 包） |
| OSQP | osqp + osqp-python |
| NumPy / SciPy | 标准科学计算栈 |
| Matplotlib | 用于结果可视化 |

物理参数（MuJoCo 默认 cart-pole）：
- 小车质量 $m_c = 1.0$ kg
- 摆杆质量 $m_p = 0.1$ kg
- 摆杆长度 $l = 0.5$ m
- 重力 $g = 9.81$ m/s²
- 控制周期 $T = 0.01$s（100 Hz，与 walking MPC 的典型 1kHz 不同，但便于观察）

---
Phase 1：LQR 基准控制器

目标：建立无约束最优控制的性能基准。

1.1 线性化动力学模型

在 upright 平衡点 $(x, \dot{x}, \theta, \dot{\theta}) = (0, 0, 0, 0)$ 附近线性化 cart-pole 动力学，得到连续时间状态空间：

$$\dot{x} = A_c x + B_c u$$

其中状态 $x = [x, \dot{x}, \theta, \dot{\theta}]^\top$，控制 $u = F$（小车推力）。

任务：手写推导 $A_c, B_c$，然后用零阶保持（ZOH）或欧拉法离散化为 $A_d, B_d$，采样周期 $T=0.01$s。

1.2 求解代数 Riccati 方程

用 scipy.linalg.solve_discrete_are 求解离散时间 ARE，得到 LQR 增益矩阵 $K$。

权重建议：
- $Q = \text{diag}(1, 0.1, 10, 0.1)$ — 优先稳定摆角，其次是小车位置
- $R = 1$ — 推力惩罚

1.3 闭环仿真验证

在 MuJoCo 中运行 5 秒仿真：

初始条件：摆杆偏离 upright 一个小角度，如 $\theta_0 = 0.1$ rad（约 5.7°），其余为 0。

记录数据：
- 状态轨迹 $x(t)$：小车位置、摆角
- 控制轨迹 $u(t)$：推力
- 代价累积 $J(t) = \sum (x^\top Q x + u^\top R u)$

验证指标：
- 摆角是否在 2 秒内收敛到 $\pm 0.01$ rad 以内？
- 小车位置是否稳定在原点附近？
- 控制量是否平滑、无高频抖动？

---
Phase 2：线性 MPC 实现

目标：在同样线性模型上实现 MPC，对比与 LQR 的跟踪性能。

2.1 稠密 QP 构建

基于离散模型 $x(k+1) = A_d x(k) + B_d u(k)$，预测时域 $N=20$（对应 0.2s 前瞻），构建稠密形式：

$$
X = \mathcal{A} x_0 + \mathcal{B} U
$$

任务：写出完整的 $\mathcal{A} \in \mathbb{R}^{80 \times 4}$ 和 $\mathcal{B} \in \mathbb{R}^{80 \times 20}$ 的构造逻辑。

2.2 MPC 目标函数化为标准 QP

目标：跟踪 upright 平衡点（参考 $x^{\text{ref}} = [0, 0, 0, 0]^\top$）：

$$
\min_U \quad \sum_{k=0}^{N-1} \|x(k) - x^{\text{ref}}\|_Q^2 + \|u(k)\|_R^2 + \|x(N)\|_P^2
$$

任务：把目标函数写成标准 QP 形式 $\frac{1}{2} U^\top H U + g^\top U$，明确写出：
- Hessian $H$ 的构造（与 $\mathcal{B}, Q, R$ 的关系）
- 梯度 $g$ 的构造（与 $x_0, \mathcal{A}, x^{\text{ref}}$ 的关系）

2.3 OSQP 求解器封装

用 osqp.OSQP() 建立求解器。注意：
- OSQP 要求问题形式：$\min \frac{1}{2} z^\top P z + q^\top z$ s.t. $l \leq Az \leq u$
- 需要把变量 $z$ 设为 $U$（控制序列），或扩展为 $[X; U]$（状态+控制序列）
- 建议：先只用 $U$ 作为优化变量（稠密形式），约束通过 $\mathcal{A}, \mathcal{B}$ 映射到状态

2.4 滚动 MPC 闭环仿真

每帧流程：
1. 读取当前 MuJoCo 状态 $x(k)$
2. 构建 QP（更新 $g$ 中的 $x_0$）
3. 调用 OSQP 求解
4. 只执行 $u^*(0)$
5. 推进仿真一步，循环

初始条件：与 Phase 1.3 相同（$\theta_0 = 0.1$ rad）。

对比实验：在同一初始条件下分别运行 LQR 和 MPC，记录相同指标。

---
Phase 3：约束对比实验（核心）

目标：验证 MPC 相对于 LQR 的核心优势——显式约束处理能力。

3.1 加入控制量约束（推力饱和）

给 MPC 加入推力限幅：
- $u_{\min} = -5$ N
- $u_{\max} = +5$ N

在 OSQP 中，这对应于对优化变量 $U$ 的直接 box constraint。

实验设计：
- 初始条件改为 $\theta_0 = 0.3$ rad（更大扰动，需要更大恢复力矩）
- 分别运行：
  1. LQR（无约束）：推力可能会饱和（超过 ±5N），观察其饱和后的性能退化
  2. MPC（有约束）：推力被硬限制在 ±5N，观察其如何在约束内规划恢复轨迹

记录重点：
- 推力是否饱和？饱和持续多久？
- 摆角超调量（overshoot）
- 稳定时间（settling time）
- 是否有 LQR 因为饱和导致失稳/摆杆倒下，而 MPC 仍能恢复的情况？

3.2 加入状态约束（可选进阶）

尝试加入摆角约束 $|\theta| \leq 0.5$ rad，观察 MPC 是否能在摆杆偏离较大时主动「限制」摆动幅度。

注意：状态约束需要通过 $\mathcal{A} x_0 + \mathcal{B} U$ 映射为对 $U$ 的线性约束。

---
Phase 4：预测时域 N 的 Trade-off 分析

目标：定量理解 $N$ 对性能与计算的影响。

4.1 N 的取值网格

在 MPC（带约束 ±5N）上测试：
- $N = 5, 10, 20, 40, 80$

4.2 记录指标

对每个 $N$，记录：

| 指标 | 记录方式 |
|------|----------|
| 单次 QP 求解时间 | time.perf_counter() 测量 OSQP solve() 调用 |
| 平均求解时间 | 5 秒仿真内所有帧的平均 |
| 最大求解时间 | 最坏情况帧 |
| 摆角收敛时间 | 从 $\theta_0 = 0.1$ 到 $|\theta| < 0.01$ 的时间 |
| 累积代价 $J$ | 与 LQR 对比 |
| 推力平滑度 | $\sum \|u(k) - u(k-1)\|^2$，衡量 jerk-like 变化 |

4.3 分析要求

基于实验数据，回答以下问题：

1. $N$ 太小（如 5）：控制器是否变得短视？摆角是否出现明显 overshoot？为什么？
2. $N$ 太大（如 80）：求解时间是否超过控制周期 $T=0.01$s？如果超过，意味着什么（实时性失效）？
3. 最优 $N$ 的范围：在你的机器上，哪个 $N$ 能在「求解时间 $< T$」的前提下给出最好的跟踪性能？
4. walking 机器人为什么用线性 MPC？ 结合你的实验：如果非线性 MPC 的单次求解时间是线性 MPC 的 10-100 倍，在 1kHz 控制周期下是否可行？
