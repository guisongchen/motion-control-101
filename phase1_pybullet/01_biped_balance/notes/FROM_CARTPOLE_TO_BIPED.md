# 从 Cart-Pole 到双足机器人

## 核心发现

通过 Cart-Pole 实验，我们验证了:

| 参数组合 | 结果 | 原因 |
|---------|------|------|
| 只有 Kp | ❌ 失败 | 纠正误差但产生振荡 |
| Kp + Kd | ✅ 成功 | 纠正 + 阻尼，稳定平衡 |

**关键公式**:
```
控制力 = Kp × 角度误差 + Kd × 角速度
```

---

## 概念映射: Cart-Pole → 双足机器人

```
Cart-Pole                    双足机器人
─────────────────────────────────────────────────
小车 (Cart)        →         脚与地面接触点
摆杆 (Pole)        →         躯干 (Torso)
摆杆角度 θ         →         躯干倾斜角 (pitch)
摆杆角速度 ω       →         躯干角速度
小车水平力 F       →         脚踝力矩 (ankle torque)
```

### 控制目标对应

| Cart-Pole | 双足机器人 | 控制方法 |
|-----------|-----------|---------|
| 保持 θ ≈ 0 | 保持躯干直立 | PD 控制 |
| 限制小车位置 | 限制质心在支撑面内 | 髋/膝协调 |
| 防止摆杆倒下 | 防止机器人摔倒 | 多关节协调 |

---

## 迁移策略

### 阶段 1: 单层控制 (类似 Cart-Pole)

只控制**脚踝**，保持躯干直立:

```python
# Cart-Pole 公式
force = Kp * torso_pitch + Kd * torso_pitch_velocity

# 映射到脚踝力矩 (注意方向!)
ankle_torque = -Kp * torso_pitch - Kd * torso_pitch_velocity
```

**为什么脚踝力矩要负号?**
- Cart-Pole: 推小车 → 带动摆杆
- 双足: 脚踝产生力矩 → 直接作用于躯干
- 方向相反，所以需要负号

### 阶段 2: 增加高度控制

Cart-Pole 没有高度控制，但双足需要:

```python
# 高度控制 → 膝盖弯曲
height_error = target_height - current_height
knee_angle = base_knee + Kp_height * height_error
```

### 阶段 3: 增加位置控制

防止机器人左右跑动 (类似 Cart-Pole 的小车位置):

```python
# 质心水平位置控制 → 髋关节
com_x_error = target_x - com_x
hip_angle = base_hip + Kp_pos * com_x_error
```

### 阶段 4: 整合控制 (完整双足)

```
控制目标          执行关节          控制律
─────────────────────────────────────────────────
躯干姿态 (pitch)   脚踝             τ_ankle = -Kp_p×pitch - Kd_p×ω
高度 (z)           膝盖             θ_knee = θ_base + Kp_h×h_error
水平位置 (x)       髋关节           θ_hip = θ_base + Kp_x×x_error
```

## 关键差异与应对

### 差异 1: 执行器数量

```
Cart-Pole: 1个执行器 (小车推力)
双足:      多个执行器 (髋、膝、踝)

应对: 分层控制
  - 姿态控制 →  ankles
  - 高度控制 →  knees
  - 位置控制 →  hips
```

### 差异 2: 接触稳定性

```
Cart-Pole: 小车在轨道上，不会滑动/分离
双足:      脚与地面接触可能滑动或分离

应对: 监测接触力，确保脚不打滑
```

### 差异 3: 初始条件

```
Cart-Pole: Gym 环境自动重置到可平衡状态
双足:      落地后可能有前倾/晃动

应对: 先让机器人自稳定几秒，再启用控制
```
